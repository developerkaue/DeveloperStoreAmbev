# DeveloperStoreAmbev

Este documento detalha a implementação do projeto de Developer Evaluation, seguindo Clean Architecture, DDD (Domain-Driven Design), SOLID, e padrões modernos de desenvolvimento.

O projeto será uma API REST para gerenciar vendas, incluindo:

Cadastro e gerenciamento de vendas e itens de venda

Aplicação de regras de desconto

Persistência em PostgreSQL

Suporte a MongoDB para leitura

Cache com Redis

Testes unitários e de integração

Publicação de eventos de domínio

Tecnologias Utilizadas

C# 12 e .NET 8.0

Entity Framework Core (PostgreSQL)

MongoDB (NoSQL)

Redis (Cache)

MediatR (CQRS, Event-Driven)

AutoMapper (Mapeamento de DTOs)

xUnit e Moq (Testes unitários e mocks)

Docker e Docker Compose

FluentValidation (Validações)
